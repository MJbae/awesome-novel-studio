# Novel Bootstrap — 부트스트랩 문서 생성 스킬

웹소설의 기초 설계 문서를 생성한다. 이 문서는 캐릭터 설계자와 플롯 설계자가 작업하는 토대가 된다.

## 작업 절차

1. `references/genre-dna-framework.md`를 읽어 장르 프레임워크를 숙지한다
2. 사용자의 소설 컨셉(직업, 분위기, 차별화 방향)을 분석한다
3. 아래 출력 템플릿에 맞춰 부트스트랩 문서를 작성한다
4. 기존 인기작과의 유사도를 점검하고 차별화 포인트를 명시한다

## 출력 템플릿

```markdown
# [작품 가제] — 부트스트랩 문서

## 1. 작품 개요
- **로그라인**: 한 줄 요약 (주인공 + 핵심 갈등 + 차별화 포인트)
- **장르 태그**: [주요 장르], [서브 장르], [추가 태그]
- **타겟 플랫폼**: [문피아/네이버시리즈/카카오페이지/리디/조아라/노벨피아 중 1개] (→ 확장 가능성)
- **예상 분량**: [총 화수] / [1회당 글자수]
- **톤 & 무드**: [분위기 키워드 3~5개]

## 2. 세계관 설정
- **시대 배경**: [서사 시작 시점] ~ [현재/미래]
- **전문 분야**: [직업군 + 구체적 세부 분야]
- **전문 분야 깊이 전략**: [전문 지식을 서사에 녹이는 방법]
- **사회적 맥락**: [해당 직업의 사회적 위치, 권력 구조]

## 3. 서사 기점 설정
### 3-1. 주인공의 과거/배경
- **최고점**: [주인공이 도달했거나 도달할 수 있었던 위치]
- **좌절/전환**: [서사를 추동하는 핵심 사건 — 비극/계기/변화]
- **감정 핵심**: [독자가 느낄 공감/분노/기대의 핵심]

### 3-2. 서사 동력
- **트리거**: [서사를 시작하게 하는 계기]
- **시작 상태**: [주인공의 서사 시작 위치/상황]
- **핵심 능력/자산**: [주인공이 가진 특별한 역량이나 정보]

### 3-3. 핵심 역량 모듈
| 시점 | 정보/능력 | 활용 방법 | 서사적 기능 |
|------|----------|----------|-----------|
| [시점] | [정보/능력] | [주인공이 활용하는 방식] | [어떤 서사 효과] |
| ... | ... | ... | ... |

## 4. 셀링포인트
- **핵심 차별화**: [기존 작품 대비 독특한 점 1개]
- **주요 쾌감 요소**: [독자가 느낄 카타르시스의 종류]
- **익숙함:새로움 비율**: 7:3 기준 어디에 새로움을 배치했는가

## 5. 플랫폼 전략
- **무료 구간**: 1~[N]화 (1권 분량)
- **유료 전환 시점**: [N]화 (투베 진입 조건 분석)
- **유료 전환 훅**: [전환 시점에 배치할 서사 이벤트]
- **일일 연재 계획**: [연재 주기, 비축분 전략]

## 6. 미디어 믹스 확장성
- **웹툰화 적합도**: [시각적 장면의 비중]
- **드라마화 적합도**: [현실 배경의 활용도]
- **핵심 시각화 장면**: [웹툰/드라마에서 빛날 장면 3개]

## 7. 기존 작품 대비 포지셔닝
| 비교 작품 | 유사점 | 차별점 |
|----------|--------|--------|
| [작품1] | [공통 요소] | [이 작품만의 차이] |
| [작품2] | [공통 요소] | [이 작품만의 차이] |
```

## 핵심 체크포인트

- [ ] 로그라인이 한 문장으로 작품의 매력을 전달하는가
- [ ] 주인공의 과거/배경이 독자의 공감/기대를 유발하는가
- [ ] 핵심 역량 모듈이 최소 10개 항목 이상인가
- [ ] 차별화 포인트가 명확하고 기존 작품과 구별되는가
- [ ] 전문 지식 깊이 전략이 "냄새만 풍기되 흐름은 끊지 않는" 수준인가
